# 多食堂多窗口仿真系统

## 项目概述
这是一个学生食堂仿真实验系统，采用分层模块化设计。系统包含6个模块：仿真控制、事件调度、食堂管理、学生行为、数据接口、前端展示。

## 模块结构
- `cafeteria_management/` - 食堂管理模块
- `student_behavior/` - 学生行为模块
- `tests/` - 单元测试

## 食堂管理模块
职责：管理多个食堂及其内部资源状态
- 多食堂支持：系统可配置N个食堂，每个食堂独立运营
- 多窗口管理：每个食堂内可配置M个服务窗口，支持并行服务
- 队列管理：每个窗口维护独立的学生排队队列，支持先进先出（FIFO）
- 座位管理：每个食堂维护就餐座位的占用/空闲状态
- 资源分配：提供智能窗口分配、座位分配算法
- 状态查询：提供实时资源状态查询接口
- 统计功能：记录各食堂/窗口的服务统计信息

## 学生行为模块
职责：管理单个学生的生命周期状态
- 状态定义：到达、选择食堂、排队中、服务中、寻找座位、就餐中、离开等状态
- 状态转换：在不同状态间转换，每个转换有明确触发条件
- 食堂选择：支持学生根据偏好、等待时间等因素选择食堂
- 窗口选择：在选定食堂内选择最优窗口排队
- 行为参数：支持不同类型学生的个性化行为（如用餐时长偏好、食堂偏好）
- 状态监控：提供当前状态查询接口

## 技术要求
- Python 3.12
- 面向对象设计，单一职责原则
- 线程安全（使用锁机制保护共享资源）
- 遵循Python编码规范（PEP 8）
